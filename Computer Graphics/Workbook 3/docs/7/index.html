SVG Exercise

Box 1: Exercise 5, SVG

Above is a big SVG that is empty. It is 7-1.html and 7-1.svg. Fill it in with an interesting picture (by editing that file).

You should make the picture by hand, i.e., edit the SVG in a text editor. Do not create or edit the picture in a picture editing tool (like Illustrator or InkScape).

Your picture should make use of hierarchical modeling. It should have groups, transformations, and instancing. There should be more than one <use> tags, as well as <g> (group) tags that have transformations attached.

To get all regular points for this exercise, it's OK if your picture only uses basic shapes. But it is also OK if you use this opportunity to learn more about SVG and use some more advanced features.

You will earn 15 points for making a picture: 5 points for making a picture with multiple parts, 5 points for using transformations (using a <g> tags), and 5 points for instancing (using <use> tags).

You can earn up to 3 bonus points (if you earn more than 3, we will only give you a maximum of 3):

  1. 1-2 points for making really great use of <use> to do something creative (with lots of replication)
  2. 1-2 points for making a particularly cool or creative picture

Note: the bonus points are at the discretion of the grader.

If you're stumped for ideas, you could try re-creating one of the Canvas pictures in the workbook (for example, the windmills picture - without the animation).

Technically, the picture in Page 5 Box 2 (the SVG example 5-2.svg) would satisfy the requirements. You can refer to this picture (you have the source code), but please make your own picture.

You should do this assignment by writing SVG. Don't use JavaScript to create the SVG. And make a static picture: while SVG has interaction and animation capabilities, usually these are done with JavaScript, and that gets away from the goals of this assignment. Do not try to figure out SVG's built in animation capabilities.

This is the last page of the workbook. Don't forget to fill in your README.md. And of course, commit and push to GitHub often!
