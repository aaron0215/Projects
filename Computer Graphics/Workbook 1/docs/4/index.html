Box 1 - Referring to HTML

We often need our JavaScript code to be able to refer to elements on the page so we can do things to them. The web browser provides many functions that allow us to search through the DOM (Document Object Model, the data structure that represents the web page). The most useful for us is to find an element by id. Remember from Page 1 (HTML and CS559), that we can label elements with an id attribute. In our JavaScript code, we can search for an element by Id and do something to it. The box has an example (and as you can guess, you should read the code in 4-1.html).

If you look at the code, you'll see a pattern: we find an element, and we do something to it. We changed the text of a span named span1, we changed the color of span2 and added a click handler to the button. Read the code and make sure you understand how it works.

There are many other ways to find elements on web pages - this is the simplest one, and the one we will use for almost everything in class.

Here's a hint for looking at the code in the debugger: if you open the inner page (4-1.html for this one), it still works in the browser. But the browser is working on this page. If you use “view source”, you'll see only the page's source. If you go to the browser console, you can also see the code - and use the debugger to set a breakpoint.

This is a good time to practice using the debugger. Put a breakpoint before the text in span1 is changed (stop on line 19). And you'll see the that HTML says something different before it is changed.

Box 2 - Button Events

We saw this on the last page, but when a button is clicked it calls the onclick handler. More generally, when something happens (to almost any element), it creates an event. We can tie a function (called a handler) to the event, and do things when the event happens. The style of programming is called event driven programming - we basically write code that does things in response to events. Even our initial set up thing happen in response to the window onload event.

Different elements have different events associated with them. Buttons actually have a several different events (onclick that we've seen, as well as onmousedown, onmouseup, and onmouseleave and several others). This box uses them in different ways to get different behaviors. Try moving the mouse while clicking. And then make sure to read the code.

An obvious, but important thing here is that our code only gets run when an event happens. Otherwise, we don't do anything. This is called event-driven programming.

Different kinds of HTML elements generate different kinds of events.
